# 候选页双模式选片

## What This Is

为 FV AutoVideo 平台的候选页设计并实现双模式选片功能，解决"总览高密度浏览"与"选片快速决策"两类任务的交互冲突。

## Core Value

让用户在大量镜头中高效完成选片：列表模式快速扫全局，选片模式键盘流快速决策。

## Context

### 项目背景

- FV AutoVideo 是视频生成与编辑平台
- 候选页当前承载所有选片任务，但布局无法同时满足"高密度浏览"和"聚焦快速决策"两类需求
- 用户在大量镜头（数十到数百个）时选片效率低下

### 已知约束

- 首版面向桌面端键盘工作流
- 平板和移动端不是首版重点
- 本期不支持批量选择
- 撤销仅支持当前会话内

## Requirements

### Active

- [ ] 实现页面双模式切换（overview / picking）
- [ ] 列表模式高密度展示所有镜头
- [ ] 选片模式单镜头聚焦布局（候选区 65-75% 视觉权重）
- [ ] 键盘 1/2/3/4 选择候选
- [ ] 键盘 ArrowLeft/ArrowRight 切换镜头
- [ ] 离开镜头时自动提交当前激活候选
- [ ] 激活候选自动循环播放
- [ ] 进度显示：当前第 X/Y + 已选 Z/Y

### Out of Scope

- 批量选择能力
- 跨刷新持久化撤销
- 移动端完整适配
- 镜头详情快捷入口（P2）

## Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| 双模式拆分 | 两类任务对界面要求冲突，强行兼顾体验差 | 列表+选片双模式 |
| 键盘优先 | 大量镜头时鼠标效率低，数字键直接选择是核心效率来源 | 1-4 数字键快速选片 |
| 选片模式自动提交 | 用户需求"不切换则默认候选为最终选择" | 离开镜头时自动提交 |
| 模式记忆粒度 | 按项目/剧集级别，不按单镜头 | 项目级模式记忆 |
