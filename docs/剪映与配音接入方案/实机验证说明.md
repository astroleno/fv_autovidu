# 剪映草稿实机验证说明

本文件配合「剪映草稿导出」功能使用。

## 为何时间轴曾显示空白？

剪映桌面版**底部时间轴**主要读取工程目录下的 **`draft_content.json`**（素材库/元数据另有其它 JSON）。  
若仅自研 **`draft_info.json`** 而不生成 **`draft_content.json`**，可能出现**能打开工程但时间轴为空**的现象。

当前 **fv_autovidu** 已集成 **pyJianYingDraft**，在导出时生成：

- `draft_content.json`（时间轴、轨道、片段）
- `draft_meta_info.json`（来自官方模板并写入 `draft_id` / `draft_name` / `tm_duration`）
- `materials/`（已复制视频与可选配音）

竖屏画布默认 **1080×1920**（9:16），与短视频常见比例一致。

## 建议步骤

1. 后端安装依赖：`pip install -r requirements.txt`（含 `pyJianYingDraft`）。
2. 在分镜板为若干镜头**选定视频**，使用 **导出 → 剪映草稿**，填写本机草稿根目录。
3. 在剪映中打开 **`{草稿根}/{draftId}/`** 工程（与 Toast 提示路径一致）。
4. 若仍异常：记录**剪映版本号**（剪映 6+ 可能对「由软件保存」的草稿加密；**本工具生成的是未加密 JSON**，一般可导入）。可向 `jianying_service._write_jianying_draft_pyjdraft` 与 pyJianYingDraft 文档对照排查。

## 排查：「不支持的视频素材类型 '.mp4'」

该文案来自 **pyJianYingDraft** 的 `VideoMaterial`：当 `pymediainfo.MediaInfo.can_parse()` 为 **False**（本机未正确安装或无法加载 **libmediainfo** 动态库）时，会抛出带 `'.mp4'` 后缀的 `ValueError`，**与 MP4 是否被支持无关**。

- **后端预检**：`jianying_service._ensure_mediainfo_for_jianying()` 会在导出前检测，并返回更明确的错误提示。
- **环境修复**：macOS 可 `brew install mediainfo`；Linux 常见 `libmediainfo0v5` 等包；装好后在同一终端执行 `python3 -c "from pymediainfo import MediaInfo; print(MediaInfo.can_parse())"` 应输出 `True`。
- **配音轨**：`AudioMaterial` 在同样条件下会报「不支持的音频素材类型 …」，根因一致。
