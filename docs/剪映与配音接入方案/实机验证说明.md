# 剪映草稿实机验证说明

本文件配合「剪映草稿导出」功能使用。

## 为何时间轴曾显示空白？

剪映桌面版**底部时间轴**主要读取工程目录下的 **`draft_content.json`**（素材库/元数据另有其它 JSON）。  
若仅自研 **`draft_info.json`** 而不生成 **`draft_content.json`**，可能出现**能打开工程但时间轴为空**的现象。

当前 **fv_autovidu** 的导出目标已收敛到一套**最小未加密草稿**，核心文件为：

- `draft_info.json`
- `draft_content.json`
- `draft_meta_info.json`
- `draft_virtual_store.json`
- `Resources/`（已复制视频与可选配音）

竖屏画布默认 **1080×1920**（9:16），与短视频常见比例一致。

## 2026-03 实机排查结论

这次联调里确认了两条关键结论：

1. **不要把“剪映自己保存后的工程目录”当作导出目标模板。**  
   当前本机剪映会在部分工程里生成 `Timelines/`、`timeline_layout.json`、`attachment_*.json`、甚至加密的 `draft_info.json` / `draft_meta_info.json`。这些文件属于**剪映保存后的内部工程形态**，不是导入所需的最小集合。

2. **当前版本先对齐“最小未加密样本”，不要额外补壳。**  
   实测可参考样本 `test0320-干湿双测-硅谷徒步极客-1774437441496/`，其结构只有：
   - `draft_info.json`
   - `draft_content.json`
   - `draft_meta_info.json`
   - `draft_virtual_store.json`
   - `Resources/`

   该样本**没有**：
   - `Timelines/`
   - `timeline_layout.json`
   - `attachment_*.json`

因此，若后续再次出现“点了没反应 / 无法打开草稿”，优先检查：

- 导出目录是否仍只包含上述最小文件集合
- `draft_info.json` 是否被错误写成“现代工程壳”结构
- `draft_meta_info.json` 是否被额外塞入非样本字段
- `Resources/` 内素材路径是否已重写为最终草稿目录的绝对路径

## 建议步骤

1. 后端安装依赖：`pip install -r requirements.txt`（含 `pyJianYingDraft`）。
2. 在分镜板为若干镜头**选定视频**，使用 **导出 → 剪映草稿**，填写本机草稿根目录。
3. 在剪映中打开 **`{草稿根}/{draftId}/`** 工程（与 Toast 提示路径一致）。
4. 若仍异常：记录**剪映版本号**（剪映 6+ 可能对「由软件保存」的草稿加密；**本工具生成的是未加密 JSON**，一般可导入）。可向 `jianying_service._write_jianying_draft_pyjdraft` 与 pyJianYingDraft 文档对照排查。

## 媒体探测：ffprobe（无需 MediaInfo）

**fv_autovidu** 在剪映导出时使用 **`jianying_ffprobe_materials`**：用 **ffprobe** 读取素材宽高与时长，再手工构造草稿素材字段，**不依赖**系统安装 **libmediainfo**。与粗剪、时长探测共用 **ffmpeg/ffprobe**，请保证本机已安装 FFmpeg 且 `ffprobe` 在 `PATH` 中。

若导出失败并提示 `ffprobe` 未找到，请安装 FFmpeg（macOS：`brew install ffmpeg`）。
