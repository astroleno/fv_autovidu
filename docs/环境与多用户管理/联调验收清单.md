## 2. 必做手工检查（建议按序勾选）

### 2.1 无 `config/feeling_contexts.json` 时降级

1. 暂时移走或重命名 `config/feeling_contexts.json`（仅本地）。
2. 重启后端，前端设置页应出现 **「仅 .env 全局」** 类提示；顶栏徽章类似 **「.env 全局」**。
3. 打开浏览器开发者工具：**非** `/api/contexts*` 的请求可仍带历史 localStorage 中的 context id（若未清除）；但 **`GET /api/contexts` 请求不应带 `X-FV-Context-Id`**（引导接口豁免），且应返回 `configured: false`。
4. 确认拉取/列表等业务在无头模式下仍走**扁平** `DATA_ROOT`（与旧行为一致）。

### 2.2 有 `feeling_contexts.json` 且 `.env` / `.env.local` 已配置 Profile 变量

1. 恢复 `feeling_contexts.json`，在 **`.env.local`**（或 `.env`）中填入 `credentialSource` 指向的变量值。
2. 设置页选择 **Profile A** → 保存后，网络请求应带 **`X-FV-Context-Id: <ProfileA 的 id>`**。
3. 切换到 **Profile B**，确认同一接口头变为 Profile B；**「验证登录」** 对当前所选 Profile 应可成功（或明确报平台/密码错误，而非 400 上下文解析失败）。
4. 顶栏环境标识与当前 Profile 的 `envKey` 一致（含生产红色样式）。

### 2.3 老数据迁移后的命名空间

1. 对一份副本数据执行 `python scripts/migrate_data_namespace.py --dry-run` 再实迁（或按你们约定目录）。
2. **文件 URL**：分镜/资产图请求路径应为 **`contextId/projectId/episodeId/...`**（有 Profile 时），浏览器 200。
3. **任务**：`tasks` 与当前 Header 的 `context_id` 过滤一致；新任务写入非空 `context_id`（可在 SQLite 或接口列表中抽查）。
4. 若仍有个别资源 404，确认后端是否命中**新路径**，以及是否仍需**旧路径回退**（迁移窗口期）。
