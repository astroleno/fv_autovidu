# 剪映字幕竖屏位置规范（本仓库「规范版」）

**状态**：与 `jianying_text_track` 中 `position_mode=jianying_spec` 对齐  
**日期**：2026-03-31

---

## 1. 经验公式（像素 Y）

在剪映竖屏工作流中，字幕纵向位置可用下列经验关系（**n 为参与公式的行数**，**1 ≤ n ≤ 3**）：

\[
Y = -100n - 400
\]

- **Y**：与剪映**属性面板里「纵向位置」与整幅画布高度相乘后得到的读数**（或与公式对照时用的像素 Y）一致；向下为负时，多行字幕更「低」对应更负的 Y。
- **n**：由「估算行数」经 **上限 3** 截断得到（见下文「行数怎么数」）；公式中 **不会出现 n>3**。

---

## 2. 与 pyJianYingDraft `transform_y` 的换算

[pyJianYingDraft](https://pypi.org/project/pyJianYingDraft/) 的 `ClipSettings` 文档称 **垂直位移 `transform_y` 的单位为「半个画布高」**；若按 \(\text{transform\_y} = Y/(H/2)\) 写入草稿，剪映界面里**与整幅高度相乘**的读数会变成 **\(2Y\)**（例如一行出现 **-1000** 而非公式 **-500**）。

本仓库与产品约定：**使界面读数与上式 \(Y=-100n-400\) 一致**，故导出时：

\[
\text{transform\_y} = \frac{Y}{H}
\]

其中 \(H\) 为**当前导出画布高度**（竖屏 1080p 一般为 **1920**，720p 竖屏一般为 **1280**）。同一文案在 720p / 1080p 下会随 \(H\) 自动缩放。

---

## 3. 行数 n 怎么取

### 3.1 字幕正文从哪来

`subtitle_text_from_shot` 仅使用：**译文** → **台词原文** → **结构化对白**。**不使用** `visualDescription`（画面描述不是台词）。

### 3.2 估算行数（再代入公式）

`estimate_subtitle_line_count(text)`（无 3 行上限）：

1. **显式换行**：按 `\n` / `\r\n` 拆行，忽略空行；非空行数 **> 1** 时，行数 = 非空行条数。
2. **无换行或整段一行**：按剪映竖屏常见密度 **估算自动折行行数**：
   - 英文：拉丁词计数（约 **6～8 词/行**，实现取 **7**）；
   - 中日韩统一表意文字：按「**每字约半词宽**」累加进等效词数，再除以 7；
   - 无词无字时按字符长度粗算。

`jianying_spec_line_count(text)` = `min(estimate_subtitle_line_count(text), 3)`，用于 \(Y=-100n-400\)。

### 3.3 与剪映实际折行的差异

估算为近似值；若需与手动换行完全一致，请在分镜台词里插入 **真实换行符**。

### 3.4 JSON 键名

`episode.json` 若含 `dialogue_translation` 等蛇形键，服务端 `Shot` 模型已用 Pydantic `AliasChoices` 兼容。

---

## 4. 规范版字号（固定 13）

与行数 **无关**：`jianying_spec` 模式下每条字幕的 `TextStyle.size` 恒为 **13**（与产品/剪映侧约定一致）。

`manual` 模式下仍使用请求体中的统一 `subtitleFontSize`（4～16）。

---

## 5. API 行为

| `subtitlePositionMode` | 行为 |
|------------------------|------|
| `manual`（默认） | 全片字幕段使用同一 `subtitleTransformY`（-1.0～0.0）与同一 `subtitleFontSize`。 |
| `jianying_spec` | 每条字幕：`jianying_spec_line_count` 得 n（1～3）→ \(Y=-100n-400\) → `transform_y`；**字号固定 13**；**忽略**请求里的 `subtitleTransformY` 与 `subtitleFontSize`（请求体仍可传，兼容旧客户端）。 |

---

## 6. 示例（1080p 竖屏，H=1920）

| 行数 n | Y（像素） | transform_y（≈） |
|--------|-----------|------------------|
| 1 | -500 | -500/960 ≈ -0.521 |
| 2 | -600 | -600/960 = -0.625 |
| 3 | -700 | -700/960 ≈ -0.729 |
