# 剪映字幕竖屏位置规范（本仓库「规范版」）

**状态**：与 `jianying_text_track` 中 `position_mode=jianying_spec` 对齐  
**日期**：2026-03-31

---

## 1. 经验公式（像素 Y）

在剪映竖屏工作流中，字幕纵向位置可用下列经验关系（**n 为字幕行数**，`n ≥ 1`）：

\[
Y = -100n - 400
\]

- **Y**：与剪映草稿中 `clip.transform.y` 所用坐标系一致的**像素位移**方向约定（向下为负时，多行字幕更「低」对应更负的 Y）。
- **n**：**显式换行**分段后的行数（见下文「行数怎么数」）。

---

## 2. 与 pyJianYingDraft `transform_y` 的换算

[pyJianYingDraft](https://pypi.org/project/pyJianYingDraft/) 的 `ClipSettings` 约定：

> **垂直位移 `transform_y` 的单位为「半个画布高」。**

设画布高度为 \(H\) 像素（竖屏 1080p 一般为 **1920**，720p 竖屏一般为 **1280**），则：

\[
\text{transform\_y} = \frac{Y}{H/2} = \frac{2Y}{H}
\]

本仓库在导出时按**当前导出画布**的 `canvas_height` 做除法，因此同一公式在 720p / 1080p 下会自动随分辨率缩放（与「半画布高」定义一致）。

---

## 3. 行数 n 怎么取

- **规范版**在服务端用 `estimate_subtitle_line_count(text)`：按 `\n` 拆行，**忽略空行**，至少为 **1**。
- **自动换行**（仅设 `auto_wrapping`、不在文案里打换行）产生的行数**无法**在导出阶段精确推算；若需按多行公式对齐，请在分镜台词中使用**真实换行**分段。

---

## 4. API 行为

| `subtitlePositionMode` | 行为 |
|------------------------|------|
| `manual`（默认） | 全片字幕段使用同一 `subtitleTransformY`（-1.0～0.0），与历史行为一致。 |
| `jianying_spec` | 每条字幕单独计算：先数行数 n → \(Y=-100n-400\) → 再换算为 `transform_y`；**忽略**请求里的 `subtitleTransformY` 数值。 |

---

## 5. 示例（1080p 竖屏，H=1920）

| 行数 n | Y（像素） | transform_y（≈） |
|--------|-----------|------------------|
| 1 | -500 | -500/960 ≈ -0.521 |
| 2 | -600 | -600/960 = -0.625 |
| 3 | -700 | -700/960 ≈ -0.729 |
