# v6 · 节奏推导与爆点密度

**状态：规划中（Draft）**
**优先级：P0**
**日期：2026-04-20**
**适用阶段：v6.0 · 叙事追剧力修复**

---

## 文档目的

v5 在"工程质量"与"叙事结构化"上已经稳定，但**商业短剧节拍**——让观众被连续拽住的追剧密度——与甲方基线差距显著。本文档定义 v6 的节奏层设计：

- **不用预设枚举**：所有节拍参数都由 `episodeDuration × genre_bias × 剧本密度`**推导**；
- EditMap 产出片级 `rhythm_timeline` 草案；
- Scene Architect 根据 KVA 实际发生秒微调爆点时间锚（对齐 05 号文档 C 选项 1）；
- Director/Prompter 按 timeline 消费并做硬门校验。

---

## 一、节奏层短板（证据）

### 1.1 与甲方节拍量化对比

| 维度 | 行业高密度节拍参考 | 甲方《边缘·第一集》实测 | v5 `leji-v5p` 实测 | 差距 |
|------|-------------------|------------------------|--------------------|------|
| 爆点总量（120s） | 3–5 小爆 + 1–2 强爆 = 4–7 个 | ~5 小爆 + 2 强爆 | `satisfaction_points` **1–2 条** | 🔴 |
| 黄金开场 0–3s | 1–2 镜，强冲突直给 | 跨步推门前置钩子 | 走廊亮相 + 传闻铺垫 | 🔴 |
| 30s 左右强反转 | 亮身份 / 掏证据 / 能力显化（三选一） | 推门撞破 + 身份差 | 改写为"职业性察觉" | 🟡 |
| 每 5s 信息密度 | 每 5s 至少 1 个新信息 | 密度高 | 多数 5s 窗口是情绪特写 | 🔴 |
| 台词字数 / 分钟 | 250–320 字 | ~580 字 / 120s ≈ 290 | `<silent>` 为主，估 < 40 字 | 🔴 |
| 小爆点五段式 | 触发-放大-扭转-兑现-余波 | 基本命中 | 多为"情绪起-情绪伏"两段 | 🔴 |
| 结尾悬念钩子 | 2–3 镜特写 + 留白 | 分屏讽刺定格 | 普通渐隐 | 🔴 |

### 1.2 根因

v5 `emotion_loops` 的五阶段命名（hook / pressure / lock / payoff / suspense）**与高密度节拍相似**，但 `completeness == partial` 也能通过软门 → 模型系统性偷懒到"半兑现"。v6 需要把"密度"与"兑现完整度"从**描述字段**升级为**可度量硬门**。

## 二、设计原则

### 2.1 模板路由 + 最小推导（v6 · 不用系数相乘）

v6 同时避开按**时长**预设（同时长不同题材差异大）与**系数相乘**（不透明、难维护）两条路，改为：按 `genre_bias.primary` **路由到预置节奏模板**（一组直接可消费的期望值参数），时间分布这类纯几何计算才用简单公式。无 brief / `confidence == low` 时**默认 `satisfaction_density_first`**（高密度爽点，v6 最安全默认）。

### 2.2 两级节拍锚

- **片级锚**：`golden_open_3s` / `major_climax` / `closing_hook` 固定三锚，任何时长都必有；
- **序列级锚**：`mini_climaxes[]` 数量、时间点由公式推导。

### 2.3 硬门 + 软门分层

| 层级 | 校验项 | 门级 |
|------|--------|------|
| 片级完整 | 三锚齐全 · major_climax 三选一（或 `null`）命中 · closing_hook 存在 | **硬门** |
| 序列密度 | `mini_climaxes` 数量 ≥ 模板下限 · 爆点间隔 ≤ 25s | **硬门** |
| 五段式完整 | `mini_climaxes[i].five_stage` 五段都有 shot 对应 | **硬门** |
| 信息点密度 | 每 5s 滑窗内至少 1 个非 none 的 `info_delta` | **硬门** |
| 对白字数 | `floor_hard`（硬拦） / `expected`（软警告） / `ceiling_hard`（硬拦防补写） | **混合** |
| 缓冲区预算 | `bonding` 总时长 ≤ `bonding_budget_sec` | 软门 |

## 三、节奏模板表与推导公式

### 3.1 输入信号

| 信号 | 来源 | 类型 | 备注 |
|------|------|------|------|
| `duration_sec` | `episodeDuration` | 数 | — |
| `genre_bias.primary` | `meta.style_inference.genre_bias.primary` | 枚举 | 缺失/low confidence → 默认 `satisfaction_density_first` |
| `block_count` | `len(block_index)` | 数 | 约束爆点数上限 |
| `source_dialogue_char_count` | `Σ(segments where type ∈ {dialogue, monologue, vo}).text.length` | 数 | **上位真相源**，覆写模板期望值 |

### 3.2 节奏模板路由（v6 冻结）

按 `genre_bias.primary` 路由，每行是一组**直接可消费**的调度参数：

| `genre_bias.primary` | `mini_per_30s` | `open_hook_type` | `major_climax_window_pct` | `dialogue_char_hint_per_min` | `bonding_budget_ratio` |
|----------------------|:--------------:|------------------|:-------------------------:|:----------------------------:|:----------------------:|
| **`satisfaction_density_first`**（**v6 默认**） | **1.3** | `conflict_direct` | 0.80 | 260 | 0.05 |
| `short_drama_contrast_hook` | 1.2 | `rumor_overheard` | 0.80 | 280 | 0.05 |
| `mystery_investigative` | 1.0 | `clue_drop` | 0.75 | 220 | 0.08 |
| `slow_burn_longform` | 0.7 | `atmosphere_open` | 0.70 | 180 | 0.12 |
| `artistic_psychological` | 0.6 | `imagery_open` | 0.65 | 140 | 0.15 |

**路由规则**：
- EditMap 读 `meta.style_inference.genre_bias.primary` 直接查表；
- 缺失 / `confidence == low` → 默认 `satisfaction_density_first`；
- 允许 `directorBrief.genre_override` 显式覆盖（用于特殊 campaign）；
- 模板参数**直接消费**，**不做系数相乘**（与 00 号文档 §九风险表对齐）。

### 3.3 `mini_climaxes` 数量与时间点

```
mini_count_hint = round(duration_sec / 30 × template.mini_per_30s)
mini_count      = min(mini_count_hint, block_count − 1)   # 每块最多承 1 个爆点

span = duration_sec − 15 − 8          # 保留头尾给 open / closing
gap  = span / (mini_count + 1)
mini_climaxes[i].at_sec = round(8 + gap × (i + 1))
```

Scene Architect 可在 `±3s` 范围内按 KVA 实际发生秒微调。

### 3.4 `major_climax` 时间点

```
major_climax.at_sec = round(duration_sec × template.major_climax_window_pct)
```

### 3.5 对白字数 · 期望 + 上下限（防目标互搏）

> **优先级仲裁（00 号文档 §0）**：原文字数是上位真相源，模板期望只是参考。稀对白剧本不得被迫补台词。

```
source_count                  = source_dialogue_char_count                          # 原文上位
episode_template_hint         = round(template.dialogue_char_hint_per_min × duration_sec / 60)
effective_expected            = round(min(source_count, episode_template_hint) × 0.95)  # 软门基线
episode_dialogue_floor_hard   = round(effective_expected × 0.5)                     # 硬下限
episode_dialogue_ceiling_hard = round(source_count × 1.05)                          # 硬上限 · 防补台词
```

三档语义：`< floor_hard` **硬拦**（疑似未消费）｜ `[floor_hard, expected)` 软警告 ｜ `[expected, ceiling_hard]` 通过 ｜ `> ceiling_hard` **硬拦**（违反 T01 对白保真）。

### 3.6 缓冲区预算

```
bonding_budget_sec = clamp(round(duration_sec × template.bonding_budget_ratio),
                            0 if duration_sec < 60 else 3, 25)
```

## 五、`major_climax.strategy` 三选一 + `null` 合法化

| 枚举 | 语义 | 必备硬元素 |
|------|------|------------|
| `identity_reveal` | 亮身份（令牌 / 制服 / 头衔） | 仰拍 + 令牌特写 + 台词重音 |
| `evidence_drop` | 掏证据（文件 / 录音 / 伤痕） | 特写递出 + 慢动作 + 对手反应 |
| `ability_visualized` | 能力显化（技能 / 金手指 / 反制手段） | 道具光效或特效镜头 + 节奏突变 |
| **`null`** | **剧本无明显强反转信号**（如偏文艺 / 生活流 / 散点叙事） | 无必备硬元素；仅保留 `at_sec` 时间锚与情绪锚 |

**选择规则**（v6 · **去掉强制回退**）：
- 剧本含"证据 / 文件 / 转账记录 / 录音" → `evidence_drop`
- 剧本含"身份差 / 制服 / 头衔 / 当众亮明" → `identity_reveal`
- 剧本含"技能 / 法术 / 金手指 / 系统" → `ability_visualized`
- 三者都无明显信号 → **`strategy = null`** + `diagnosis.notice_msg += "major_climax_strategy_unresolved"`

> **仲裁铁律（00 号文档 §0）**：禁止系统为了凑节拍而补造"更像短剧"的证据戏。`strategy = null` 时 `major_climax_strategy_lock` 硬门**跳过**，pipeline 放行，由 Director 在叙事层按原剧本收束。

## 六、Director / Prompter 硬门分配

### 6.1 Director v6 新增硬门

| 硬门 | 定义 |
|------|------|
| `info_delta_coverage` | 每 5s 滑窗内至少 1 个 shot 的 `info_delta` 非 `none` |
| `five_stage_slot_fill` | `rhythm_timeline.mini_climaxes[i].block_id` 所在 block 的 shot 数 ≥ 5，且每个 stage 都有 shot_idx 对应 |
| `major_climax_strategy_lock` | `strategy != null` 时 · `major_climax.block_id` 所在 block 必须出现该 strategy 的"必备硬元素"（正则/关键词扫描）；**`strategy == null` 时本硬门跳过** |
| `closing_hook_existence` | `closing_hook.block_id` 所在 block 必须含 `freeze_frame` 或 `split_screen` 的 shot 描述 |

### 6.2 Director 每 shot 新增字段

```jsonc
{
  "shot_idx": 1, "duration_sec": 2, "shot_size": "medium_close",
  "frame_description": "…", "dialog": "…",
  /* v6 新增 */
  "info_delta": "identity",
  "five_stage_role": { "mini_climax_seq": 1, "stage": "trigger" }
}
```

`info_delta` 枚举：`identity / motion / relation / prop / dialogue / setting / none`。

### 6.3 Prompter v6 新增铁律

```text
【铁律 17（v6 · 对白字数三档 · 防目标互搏）】
- 本 block [DIALOG] 累计字数按 block.duration 从 info_density_contract 派生三档：
  · < floor_hard       → 硬门拦截（疑似 segments 未消费或被误转 silent）
  · ∈ [floor_hard, expected)  → 软门 warning: dialogue_below_expected
  · > ceiling_hard     → 硬门拦截（疑似模型自行补台词，违反 T01 对白保真）
- 原剧本稀对白时 effective_expected 由 source 覆写下压，禁止补造对白。

【铁律 18（v6 · 爆点镜头强化）】
- strategy != null 时：major_climax 所在 block 的 shot 描述必须命中 strategy.must_shots；
- strategy == null 时：本铁律的 strategy 子项跳过，仅保留情绪锚要求；
- closing_hook 所在 block 的末 shot 必须包含 freeze_frame 或 split_screen 描述；
- 违反 → 硬门拦截。

【铁律 19（v6 · 缓冲区预算）】
- 所有 psychology_plan.group == "bonding" 的 block 时长之和 ≤
  rhythm_timeline.info_density_contract.bonding_budget_sec；
- 超出 → 软门 warning，pipeline 建议合并或删除部分 bonding block。
```

## 七、Pipeline 校验层

```python
# 伪代码：节奏层校验（v6 · 三档对白 + strategy=null 兼容）
def check_rhythm_timeline(edit_map, director_outputs, prompter_outputs):
    rt = edit_map["meta"]["rhythm_timeline"]
    idc = rt["info_density_contract"]
    dur = rt["derived_from"]["duration_sec"]
    hard, warn = [], []

    # 1) 片级三锚完整（strategy 允许为 null）
    for key in ("golden_open_3s", "major_climax", "closing_hook"):
        if not rt.get(key):
            hard.append(f"rhythm_timeline.{key} missing")

    # 2) mini_climaxes 数量下限（走模板路由）
    expected = derive_mini_count_from_template(rt["derived_from"],
                 block_count=len(edit_map["appendix"]["block_index"]))
    if len(rt["mini_climaxes"]) < expected:
        hard.append(f"mini_climaxes={len(rt['mini_climaxes'])} < expected={expected}")

    # 3) 爆点间隔上限（> 25s → warning）
    points = sorted([mc["at_sec_final"] for mc in rt["mini_climaxes"]]
                    + [rt["major_climax"]["at_sec_final"]])
    for a, b in zip(points, points[1:]):
        if b - a > 25:
            warn.append(f"rhythm_gap_exceeds_25s [{a},{b}]")

    # 4) 信息点密度：每 5s 滑窗不得空
    deltas = aggregate_info_deltas(director_outputs)
    for ws in range(0, dur, 5):
        hits = [d for (s, d) in deltas if ws <= s < ws + 5 and d != "none"]
        if not hits:
            hard.append(f"info_delta_void_window [{ws},{ws+5})")

    # 5) strategy=null 时跳过 strategy 签名校验
    strategy = rt["major_climax"].get("strategy")
    if strategy is not None:
        if not has_strategy_signature(director_outputs, strategy):
            hard.append(f"major_climax_strategy_signature_missing: {strategy}")

    # 6) 对白字数三档（floor_hard / expected / ceiling_hard 均按 block.duration 比例派生）
    for bo in prompter_outputs:
        chars = count_dialog_chars(bo)
        scale = bo["block_duration"] / dur
        floor_h   = idc["episode_dialogue_floor_hard"]   * scale
        expect    = idc["effective_expected"]            * scale
        ceiling_h = idc["episode_dialogue_ceiling_hard"] * scale
        if chars < floor_h:
            hard.append(f"dialogue_below_floor {bo['block_id']}: {chars}<{floor_h:.0f}")
        elif chars < expect:
            warn.append(f"dialogue_below_expected {bo['block_id']}: {chars}<{expect:.0f}")
        if chars > ceiling_h:
            hard.append(f"dialogue_over_ceiling {bo['block_id']}: "
                        f"{chars}>{ceiling_h:.0f}（疑似自行补台词）")

    return hard, warn
```

## 九、回归指标

基于同一剧本重跑 `leji-v5p`，预期：

| 指标 | v5 实测 | v6.0 目标 |
|------|---------|-----------|
| `mini_climaxes.length` | — | ≥ 模板下限（`short_drama_contrast_hook` / 120s → ≈ 5） |
| 每 5s 信息点空窗 | 存在（多数） | 0 个 |
| 整集对白字数 | < 40 字 | ∈ [`floor_hard`, `ceiling_hard`]，期望 ≥ `effective_expected` |
| `major_climax.strategy` 命中（`strategy != null` 时） | 未命中 | 100% |
| `major_climax.strategy == null` 占比 | — | 剧本无信号时允许，但需 `diagnosis.notice_msg` 标记 |
| `closing_hook` 含 freeze/split_screen | 0 | 100% |
| 爆点间隔最大值 | > 60s | ≤ 25s |

## 十、降级开关

| 开关 | 作用 |
|------|------|
| `--skip-rhythm-timeline` | 不产 rhythm_timeline（完全降到 v5 行为） |
| `--rhythm-soft-only` | 把所有节奏硬门降为 warning（排查期使用） |
| `--dialogue-floor-ratio=<float>` | 覆盖 `floor_hard = expected × ratio` 中的 ratio（默认 0.5） |
| `--dialogue-ceiling-ratio=<float>` | 覆盖 `ceiling_hard = source × ratio` 中的 ratio（默认 1.05，放宽到 1.15 以适配大量口头语填充） |
| `--genre-template=<name>` | 强制路由到指定模板，忽略 `genre_bias.primary`（回归对比使用） |

## 十一、小结

- **模板路由替代系数相乘**：按 `genre_bias.primary` 查表，不做 `genre_factor × density_factor`；无 brief 默认 `satisfaction_density_first`；
- **两级锚点**：片级三锚（open / major / closing）必有，`mini_climaxes` 数量/时间由模板 + 几何公式推；
- **`strategy = null` 合法化**：剧本无强反转信号时不强行补造证据戏，硬门跳过；
- **对白字数三档**：`expected = min(source × 0.95, template_hint)`，杜绝"为凑字数补台词"，同时设 `ceiling_hard = source × 1.05` 防幻觉扩写；
- **与 00 号文档 §0 仲裁铁律对齐**：原文 > 保真 > 节奏 > 美学，所有节奏硬门都让位给上位合同。
