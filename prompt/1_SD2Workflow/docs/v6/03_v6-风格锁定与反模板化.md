## 一、问题回顾

审计发现三类"非叙事缺失"但影响交付质量的问题：

1. **调性跑偏**：甲方要的是"短剧反差钩子"，模型按"真人电影 = 文艺心理片"的默认风格惯性产出。
2. **清洁化倾向**：模型对调情/出轨桥段自动脱敏，把肢体戏改写为权力博弈对白。
3. **模板化疲劳**：`喉结滑动 / 瞳孔收缩 / 睫毛颤动 / 指节泛白 / 镜片反光` 5 个词整集重复 47 次。
4. **情节幻觉**：整集注入 3 处原剧本不存在的情节（如"调走原始影像"）。
5. **结构性构图缺失**：分屏 / 闪回 / 切镜这些结构化提示散落在文本里，模型选择性忽略。

## 二、T04 · genreBias 调性信号

### 2.1 目标

让 `renderingStyle`（视觉风格）和 `genreBias`（剧情张力风格）**正交**：

- `renderingStyle`：管"画面渲染"（真人电影 / 3D 动画 / 赛博朋克 / 古风水墨）
- `genreBias`：管"剧情张力密度"（短剧反差钩子 / 长剧慢热 / 爽点密度优先 / 悬疑心理 / 文艺心理）

### 2.2 入参 schema 扩展

`edit_map_input.json` 新增字段：

```json
{
  "renderingStyle": "真人电影",
  "motionBias": "balanced",
  "genreBias": {
    "primary": "short_drama_contrast_hook",
    "secondary": ["satisfaction_density_first"],
    "tone_anchors": {
      "beat_to_beat_reversal": true,
      "dialogue_density": "high",
      "physical_action_density": "high",
      "emotional_restraint": "low"
    }
  }
}
```

枚举值（后续可扩）：

| `genreBias.primary` | 说明 | 适用剧型 |
|---------------------|------|----------|
| `short_drama_contrast_hook` | 短剧反差钩子，强调每 10–15 秒一个情绪反转 | 家庭伦理、霸总、重生复仇 |
| `long_drama_slow_burn` | 长剧慢热，注重人物弧线 | 正剧、年代剧 |
| `satisfaction_density_first` | 爽点密度优先，反派必惨、主角必爽 | 打脸文、逆袭 |
| `mystery_psychological` | 悬疑心理，克制留白 | 悬疑、刑侦 |
| `artistic_psychological` | 文艺心理，微表情与氛围 | 文艺片、独立电影 |

> 默认值：若未显式指定，Normalizer 检测到 `scriptContent` 中含有"反转 / 爽点 / 打脸 / 绿茶 / 渣男"等标签词，则默认 `primary = short_drama_contrast_hook`。

### 2.3 Director 切片扩展

新建切片 `4_KnowledgeSlices/editmap/v6_genre_tone_anchors.md`（全量静态挂载），给出每种 `genreBias.primary` 的**正反对照**：

```markdown
## genreBias = short_drama_contrast_hook

### 正面示例（必须这样写）
- 许倩跨坐赵凯腿上，指尖挑逗
- 赵凯接电话时母亲 VO 字字扎心
- 秦若岚门外抚腹幻想 / 室内赵凯撕扯黑丝 分屏定格

### 反面示例（禁止这样写）
- 许倩端坐办公桌对面，姿态谦恭
- 赵凯平静翻阅流程文件
- 秦若岚站在走廊默默凝视虚空

### 对白密度
- 每 Block 至少 1 段原文对白（>= 8 字）
- 整集对白段数 ≥ segments 中 dialogue/monologue/vo 总数的 95%

### 微表情配额
- 单 Block 微表情词使用 ≤ 2 次
```

### 2.4 Director 主提示词锚点

在 Director v6 提示词最前面加入：

```text
【调性锚点】本集 genreBias.primary = {{primary}}
- 若 primary = short_drama_contrast_hook：禁止将调情戏清洁化，禁止将对白哑剧化，禁止用大量微表情替代肢体动作
- 若 primary = artistic_psychological：允许微表情密度高，允许留白
- 你在本集的每个 Block 的风格判断都必须对齐 primary 指引
```

## 三、T05 · 脱敏/清洁化抑制

### 3.1 目标

禁止把原剧本里的**亲密接触 / 出轨 / 调情 / 挑逗**戏改写为"权力博弈""职场对白""无声肢体"。

### 3.2 Director/Prompter 铁律 14（v6 新增）

```text
【铁律 14（v6 · 反清洁化）】
- 原剧本中以下类别的动作/台词，必须保留原语义与原肢体接触层级：
  · intimate_betrayal：跨坐 / 抱入怀 / 捏下巴 / 摸肚子 / 亲唇 / 解扣
  · performative_affection：整理衣领 / 抓住手 / 挤笑 / 反手牵手
  · dirty_dialogue：带暧昧语气的台词（如"凯哥""宝贝儿"）
- 禁止以下清洁化改写：
  · 把"跨坐"改为"坐在对面"
  · 把"解开白大褂扣子"改为"递过文件"
  · 把调情对白改为工作流程对白
  · 把调情场景整段替换为"两人保持专业距离"
- 若确实需要脱敏（平台内容安全要求），必须：
  1. 在 director appendix.desaturation_log[] 中声明原 seg_id + 脱敏理由
  2. 保留情感张力与肢体接触的暗示（如"指尖擦过手腕"）
  3. 不得整段删除或整段替换为无关情节
```

### 3.3 Director 切片新增：反清洁化正反例库

`4_KnowledgeSlices/director/v6_anti_cleansing_examples.md`：

```markdown
## 反清洁化正反例

### 例 1：许倩跨坐腿上
- 原剧本：许倩跨坐在赵凯腿上，手指在他胸口挑逗
- ✅ 正确消费：许倩跨坐赵凯腿上，左手拨弄他衬衫领口纽扣，右手沿颌线滑动

- ❌ 错误消费（权力博弈化）：许倩端坐对面，"赵院，您那票…"（审计中真实存在的 B06 写法）

### 例 2：赵凯摸肚子
- 原剧本：赵凯抚摸许倩的肚子"你才是我的宝贝儿，还有你肚子里的孩子"
- ✅ 正确消费：赵凯左手覆在许倩小腹，指腹轻压，[DIALOG] 赵凯：你才是我的宝贝儿，还有你肚子里的孩子
- ❌ 错误消费：赵凯目光落在许倩腹部方向，眼神复杂
```

## 四、T06 · 微表情去重

### 4.1 目标

- 单 Block 内同一类微表情词 ≤ 2 次
- 整集合计 ≤ 12 次（覆盖 5 个常见词：喉结滑动 / 瞳孔收缩 / 睫毛颤动 / 指节泛白 / 镜片反光）

### 4.2 Prompter 输出后扫描

Pipeline 在 `call_sd2_block_chain_v6` 里做词频扫描：

```python
MICRO_EXPR_PATTERNS = [
    r"喉结(?:上下)?(?:滑动|滚动)",
    r"瞳孔(?:微)?(?:缩|收缩)",
    r"睫毛(?:剧烈)?(?:颤动|轻颤)",
    r"指节(?:微微)?泛白",
    r"镜片(?:一闪|反光)",
]

def scan_micro_expr(prompter_outputs: list[dict]) -> dict:
    per_block: dict[str, int] = {}
    total = 0
    hits: list[dict] = []
    for block in prompter_outputs:
        count = sum(len(re.findall(p, block['sd2_prompt'])) for p in MICRO_EXPR_PATTERNS)
        per_block[block['block_id']] = count
        total += count
    return {
        'per_block': per_block,
        'episode_total': total,
        'single_block_over_cap': [bid for bid, c in per_block.items() if c > 2],
        'episode_over_cap': total > 12,
    }
```

### 4.3 输出字段（软门）

`sd2_final_report.json` 新增：

```json
{
  "micro_expr_audit": {
    "per_block": { "B01": 3, "B02": 4, "...": "..." },
    "episode_total": 47,
    "single_block_over_cap": ["B01", "B02", "B03", "B04", "B06", "B08", "B09"],
    "episode_over_cap": true,
    "pass": false,
    "warnings": ["微表情过度使用：整集 47 次 > 上限 12；建议用动作/对白替代"]
  }
}
```

### 4.4 Director 切片提示（正向引导）

`4_KnowledgeSlices/director/v6_micro_expr_budget.md`：

```markdown
## 微表情预算
- 单 Block 内，喉结/瞳孔/睫毛/指节/镜片 五词合计 ≤ 2 次
- 超额时，优先用"动作 + 后果"替代：
  · 不写"喉结滑动" → 写"赵凯打开录音笔又合上"
  · 不写"瞳孔收缩" → 写"秦若岚目光落在诊断书封皮烫金字"
  · 不写"睫毛颤动" → 写"许倩抬手拨开垂下的发丝"
```

## 五、T07 · 情节幻觉抑制

### 5.1 目标

禁止 Director 生成**原剧本中不存在**的情节钩子（如 B10 的"调走原始影像"、B06 编造的"秦氏集团提名路径"）。

### 5.2 Director 自检字段

Director appendix 新增：

```json
{
  "extra_plot_injection_check": {
    "injected_plot_points": [
      {
        "block_id": "B10",
        "shot_index": 6,
        "injected_text": "她刚调走全部原始影像",
        "reason": "本条在 normalized_script_package.beat_ledger 全文中未找到匹配",
        "severity": "high"
      }
    ],
    "injection_count": 1,
    "pass": false
  }
}
```

### 5.3 自检规则（写进 Director prompt）

```text
【自检 · 无幻觉约束】
在输出 markdown_body 之前，对每个 Block 做以下自检：
1. 列出本 Block 出现的所有"具体情节点"（人物名 + 动作 + 事件）
2. 回查 normalized_script_package.beat_ledger 中当前覆盖的 beats
3. 若某个情节点在 raw_excerpt 和 core_action 中都无法追溯，记入
   extra_plot_injection_check.injected_plot_points[]
4. 若 severity=high（事件级幻觉，如新增角色、新增文档、新增动作线）→ 必须删除该情节点
5. 若 severity=low（氛围补充，如"远处广播声"）→ 允许保留但记入日志
```

### 5.4 Pipeline 扫描（二次校验）

Pipeline 在 Director 输出后做"命名实体抽取"，比对 Normalizer 中的 `character_registry + prop_states + scene_states`：

```python
def detect_hallucinated_entities(director_output: dict, normalized: dict) -> list[str]:
    known_entities = set()
    known_entities.update([c['canonical_name'] for c in normalized['character_registry']])
    known_entities.update(normalized['state_ledger']['prop_states'].keys())
    known_entities.update(normalized['state_ledger']['scene_states'].keys())

    mentioned = extract_entities(director_output['markdown_body'])
    return [e for e in mentioned if e not in known_entities and is_concrete_entity(e)]
```

返回的实体列表会被写入 `extra_plot_injection_check.hallucinated_entities[]`。
