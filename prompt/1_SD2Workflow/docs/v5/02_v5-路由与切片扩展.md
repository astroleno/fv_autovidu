# SD2Workflow v5 · 路由与切片扩展（injection_map v2.0）

**状态：规范（Normative）**
**日期：2026-04-16**

本文件定义 v5 对 `injection_map.yaml` 的升级（v1.0 → v2.0），以及配套**微切片**的拆分方案。

> **依赖**：所有字段定义以 `07_v5-schema-冻结.md` 为准。本文件 JSON 示例**不再重复定义** `routing` 结构，只引用。

---

## 一、设计理念（v5 细化）

```mermaid
flowchart LR
    A[EditMap appendix<br/>block_index[i].routing] --> B[编排层查 injection_map v2.0]
    B --> C{按维度匹配}
    C -->|structural| D[Director always + conditional 切片]
    C -->|satisfaction motif| D
    C -->|psychology group| D
    C -->|shot_hint category| D
    C -->|paywall_level| D
    C -->|aspect_ratio<br/>(meta.video)| E[Prompter 条件切片]
    D --> F[Director 本 block 提示]
    E --> G[Prompter 本 block 提示]
```

**v5 关键升级**：把 P1/P2 切片全部**微切片化**，让 `truncated[]` 保持在 ≤ 3% 的验收区间。

## 三、token 预算（v5 重算）

### 3.1 Director 侧预算

v5.0 将 Director 上限从 **2000 → 3000**。

**最坏命中路径（所有 conditional 均命中，实际罕见）**：

| 层 | 切片 | tokens（上限） |
|----|------|--------------|
| always | structure_constraints | 500 |
| always | status_visual_mapping | 500 |
| conditional | structure_fewshot | 800 |
| conditional | satisfaction motif 切片（**单个命中**） | 250 |
| conditional | psychology group 切片（**最多 2 组**） | 300 × 2 = 600 |
| conditional | shot_codes category 切片（**最多 2 类**） | 200 × 2 = 400 |
| conditional | paywall level 切片（**仅尾 block 1 份**） | 400 |
| **最坏合计** | | **3450** |

- 最坏 3450 仍会触发截断。**按 priority 降序截断**后落在 3000 内（通常被截掉 paywall 或 shot_codes 中优先级较低的）。
- **典型命中**（非尾 block、1 个 motif、1 个 psy group、1 个 shot cat）：`500+500+800+250+300+200 = 2550` → 无截断。

## 六、`psychology_group` 派生字段 + 同义词兜底（v5.0 HOTFIX）

EditMap 在 appendix 里写的是 `psychology_plan[].group` 与 `psychology_plan[].effects[]`。**编排层**在查 `injection_map` 前派生一个 block 级字段：

```js
// 派生逻辑（在编排层）
block.routing.psychology_group = (meta.psychology_plan || [])
  .filter(p => p.block_id === block.block_id)
  .map(p => p.group)[0] || null;
```

- 该派生字段不进 `07` schema 的正式字段，只在编排层内存里使用。
- 切片匹配只用派生字段，LLM 侧无感知。

### 6.1 宽松策略：LLM 自由选词 + 同义词层兜底

v5.0 允许 LLM 在 `psychology_plan[].group` 上写**合法枚举之外**的自由词（如 `pressure / reversal / emotion / masking / cliff` 等）。编排层通过 `psychology_group_synonym_map` 把自由词映射到 6 个合法槽后再查 `injection_map`：

```
LLM 自由词                              映射结果（合法槽）
─────────────────────────────────────────────────────────────
opening / cold_open / attention / intro          →  hook
pressure / stakes / tension / suspense /
curiosity / masking / concealment                 →  retention
reversal / twist / revelation / catharsis /
release / climax / resolution                     →  payoff
emotion / empathy / vulnerability /
intimacy / warmth                                 →  bonding
conflict / power_dynamic / confrontation /
alliance / rivalry                                →  relationship
cliff / cliffhanger / cta / hook_next             →  conversion
```

**完整权威表见 `07_v5-schema-冻结.md` §五** `psychology_group_synonym_map`（单一真相源）。

### 6.2 兜底审计

- 自由词命中同义词层 → `routing_warnings[]` 写一条 `severity: "info"`，`code: "psychology_group_synonym_fallback"`，`expected.mapped_to` 为合法槽。
- 自由词未命中同义词层 → 不注入 psychology 切片；写 `severity: "warn"`，`code: "psychology_group_synonym_fallback"`，`expected.mapped_to: null`。

> **为什么不直接硬限 enum？** 心理学命名学派多、剧本类型杂，强制收敛反而丢失信息。宽松 + 兜底 = LLM 空间大 + pipeline 稳。

## 七、切片拆分内容边界（防止重复/漏项）

### 7.1 `satisfaction/*`（T05）

每份切片只写对应母题的 **母题定义 + 2 个触发器例 + 2 个兑现画面**。**不要**把 4 个母题对比表塞进单份。

### 7.2 `psychology/*`（T06）

每份切片按"功能组视角"组织：
- 定义：本组的任务（钩、留、兑、绑、关、转）
- 武器 2–3 个：该组常用的 2–3 个心理学效应 × 每个"在 block 里怎么体现"
- 禁忌 1–2 条

不要把 12×6 矩阵塞进单文件。

### 7.3 `shot_codes/*`（T07）

每份切片只写对应大类 4 个编号的字典 + 组合禁忌（如 A 类不要连 3 个 A1）。

### 7.4 `paywall/*`（T12）

每份切片只写对应 level 的三要素清单 + 1 个反例。
