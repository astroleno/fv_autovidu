## 一、脱敏基本原则

### 1.1 三条红线

1. **不出现任何外部来源的名字**，包括但不限于：项目名、仓库名、作者代号、IP 代号、角色代号、内部工具链名。
2. **不整段抄录外部原文**。允许「借鉴理念」，禁止「复制粘贴」。
3. **不携带路径指纹**（如 `/Users/.../docs/...`、`reference/character_prompt/xxx`）出现在任何 prompt / 切片正文里；这些路径只能出现在内部 docs 的「来源追溯表」中，且须用代号替代名字。

### 1.2 参考源代号表

v5 全链路文档与切片统一使用下列代号。正文必须用代号；代号与实际来源的映射**仅**出现在本文件第三节的「追溯表」中，并且该表严格限制在内部 docs/v5 内，不进 prompts、不进切片、不进外发材料。

| 代号 | 性质 | 我们从中吸收的核心能力 |
|------|------|-----------------------|
| **参考源 A** | 古装/悬疑/玄幻向镜头执行手册（单文本） | 时序节奏（Hook/转折/高潮点位）、镜头优先级链、题材子类规则 |
| **参考源 B** | 三对话协作链竖屏短剧体系（多文件体系） | 「脑子/镜头/分镜」阶段切分、声画分离、尾部校验块、session_id 隔离、group skeleton 锚定 |
| **参考源 C** | 短剧情绪 / 心理学 / 模块化 / 爽点 / 用户历程研究包（5 份研究文档） | 情绪三曲线、心理学六功能组、七层模块、爽点三层模型、A-F 用户历程 |

> **代号用法示例（✅ 正确）**：`// 本切片源自参考源 C 的情绪曲线模型，已做术语对齐。`
> **禁止写法（❌ 错误）**：`// 参考骆门第 35 行 / fengxing-video 的共享规则 / docs/短剧/心理学视角.pdf`。

### 3.2 原文摘录的三重处理

引用外部原文时必须满足（任一条件不满足则拒绝入库）：

- **字段名**：全小写 + 下划线。例：`status_curve`、`info_gap_ledger`、`protagonist_shot_ratio`。
- **枚举值**：小写下划线。例：`status_curve.position ∈ {"up","mid","down"}`。

## 五、知识切片通用模板（v5 强制）

每份 v5 新增切片 `.md` **必须**具有下列六个小节且顺序固定；小节缺失不予合入。

```markdown
# {slice_id}

<!-- 脱敏声明：本切片源于参考源 X 的 YY 理念，经本项目改写，仅限内部使用。 -->

## 1. 目的
本切片在 Director/Prompter 阶段起什么作用，覆盖哪些路由标签。

## 2. 注入触发条件
路由标签命中逻辑（配合 injection_map.yaml 中的 match 块）。

## 3. 受控词表引用
引用本规范第 4.3 节的哪些枚举，切片内部不得新增同义词。

## 4. 内容骨架
<!-- 不得整段抄外部原文，必须是我们的话术 -->
- 要点 1 ...
- 要点 2 ...

## 5. Director/Prompter 如何消费
- 输入：...
- 使用方式：...
- 产出里的痕迹（可观察字段）：...

## 6. 反例（禁止的写法）
- 禁止 X
- 禁止 Y
```

> 该模板参考「最小切片 + 最大可审计性」原则，配合 02 文档的 `injection_map v2.0` 使用。

### 5.1 editmap/ 切片模板特例（v5.0 追加）

位于 `4_KnowledgeSlices/editmap/` 下的 6 份编剧方法论切片**不走 injection_map 路由**，改用静态拼接机制（详细机制见 `docs/v5/08_v5-编剧方法论切片.md`）。因此 editmap/ 切片的第 2 小节标题必须改为：

```markdown
## 2. 加载方式（静态挂载 · 非 injection 机制）
```

内容固定三段声明：
1. **静态挂载机制**（`fs.readdirSync('editmap/')` + 拼接到 EditMap system prompt）；
2. **为什么不走 injection_map**（EditMap 是路由器，路由器不能被自己路由）；
3. **本切片的方法论定位 + 相对其他 editmap/ 切片的职责边界**；
4. 末尾引用 `docs/v5/08_v5-编剧方法论切片.md §三` 声明加载顺序与 token 硬限（≤ 12,000）。

HTML 头部元信息必须为：

```html
<!-- 消费者：EditMap（编剧方法论层 · 静态挂载） -->
<!-- 加载方式：static_concat（call_editmap_sd2_v5.mjs 拼入 system prompt · 不走 injection_map 路由） -->
```

其余 §1 / §3 / §4 / §5 / §6 五个小节保持通用模板结构。

   肖申克|Shawshank|泰坦尼克|Titanic|哈利·?波特|Harry\s?Potter|
