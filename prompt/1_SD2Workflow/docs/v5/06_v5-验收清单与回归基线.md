## 三、v5 静态挡板正则集合（CI 使用）

下列正则由 CI 在每次 PR 上运行；**匹配数 > 0 即失败**。

> **扫描范围以 `01_v5-知识库清洗与脱敏规范.md` §3.1 为唯一真相源**。本文件不重复定义；所有正则默认按该范围扫描。

### 3.1 脱敏挡板（见 `01_` §3.1）

```
pattern_sensitive = \
  (骆门|fengxing[-_ ]?video|fengxing|Azure|reference/character_prompt|短剧剧本视效优化|镜头语言对话SOP|SOP_vertical_9x16|docs/短剧)
```

### 3.2 时长口径挡板（T01）

```
pattern_duration_legacy = \
  (5[\s-]*16|[\s\(]≥\s*12s|group_time_min|group_time_max)
```

### 3.3 路由字段挡板（T02）

```
pattern_routing_legacy = \
  (structural_tags|meta\.routing\.aspect_ratio|meta\.routing\.structural_tags)
```

> 旧字段 `structural_tags` 仅允许出现在 `scripts/sd2_pipeline/normalize_edit_map_sd2_v5.mjs` 中**紧邻** `// v5 兼容层` 注释的代码行（白名单行，见 `01_` §3.1 表）。

### 3.4 竖屏禁写项（T10，Prompter 产出抽扫）

```
pattern_vertical_forbidden_frame = \
  (镜头旋转|镜头\s*90°|斜构图|横摇|5\s*人\s*横|多人横排)
```

扫描位置：Prompter 产出 JSON 的 `sd2_prompt.[FRAME]` 段（回归阶段运行，不是 CI 静态）。

### 3.5 四段切挡板（T11，Prompter 硬门）

每个 shot 的 `sd2_prompt` 必须命中四段正则（多行模式，非贪婪）：

```
^\[FRAME\].+?^\[DIALOG\].+?^\[SFX\].+?^\[BGM\].+?
```

缺段即失败 → retry → 丢弃。

### 3.6 BGM 禁具名挡板（T11，Prompter 硬门）

```
pattern_bgm_named = \
  (^\[BGM\].*?(钢琴|吉他|弦乐|交响|周杰伦|[A-Z][a-z]+\s[A-Z][a-z]+))
```

`[BGM]` 只能是受控方向词：`tension / release / suspense / bond / none`。

## 四、回归剧本集（冻结 3 套）

v5 发布前冻结 3 套回归剧本，放在仓库内部不外发：

| 剧本代号 | 题材 | 画幅 | 预期 block 数 | 主要覆盖 | 位置建议 |
|---------|------|------|--------------|---------|----------|
| R-UrbMy | 都市悬疑 | 9:16 | 8–10 | T03/T05/T06/T07/T08/T10/T11/T12 | `tests/regression/R-UrbMy/` |
| R-CosXf | 古装仙侠 | 16:9 | 10–12 | T03/T05/T06/T07/T09/T11/T12 | `tests/regression/R-CosXf/` |
| R-EmoLf | 当代情感 | 9:16 | 6–8 | T04/T09/T10/T11（soft paywall） | `tests/regression/R-EmoLf/` |

每套剧本应包含：

```
tests/regression/R-xxx/
├── script.md              # 输入剧本（脱敏、自撰，不来自外部）
├── assets.yaml            # 人物/道具/场景资产表
├── expected_appendix.json # 期望字段骨架（允许容差；对齐 07 schema）
├── baseline_routing.yaml  # 基线命中的切片列表（v5 首跑后冻结）
└── README.md              # 该回归覆盖哪些任务
```

### 4.1 回归跑法

```
# 伪代码
for script in [R-UrbMy, R-CosXf, R-EmoLf]:
    run SD2Workflow v5 pipeline on script
    diff output appendix vs expected_appendix (tolerances: §五)
    diff routing_trace.applied[] vs baseline_routing
    run §三 正则挡板（按 01_ §3.1 范围）
    verify hard gates green; count soft warnings
```

### 4.2 回归发布门槛（v5.0）

- **GA 条件**：3 套剧本全部通过 **5 条硬门** + 软门 warning 总数 ≤ 剧本数 × 3。
- **Preview 条件**：R-EmoLf 通过 + 其余 1 项**软门**过多（仅 warning） = 允许标"预览版（T08/T09 自检未达标）"但不阻塞 GA。
- **阻塞 GA**（红线）：
  - T01 / T02 / T11 任一硬门失败；
  - 脱敏挡板（§3.1）有命中；
  - `routing_schema_valid` 在任一剧本失败。

      { "slice_id": "structure_constraints",          "tokens": 430, "from": "director.always" },
      { "slice_id": "status_visual_mapping",          "tokens": 460, "from": "director.always" },

      { "slice_id": "shot_codes.A_event",             "tokens": 220, "from": "director.conditional" },

### 6.1 `routing_warnings[]` 审计口径（v5.0 HOTFIX）

`sd2_routing_trace.json` 顶层**必须**携带 `routing_warnings[]`（与 `meta.routing_warnings[]` 一致）：

```jsonc
{
  "routing_trace": [ /* per-block applied/truncated */ ],
  "routing_warnings": [
    {
      "code":     "shot_count_budget_exceeded",
      "severity": "warn",
      "block_id": null,
      "actual":   39,
      "expected": { "target": 30, "tolerance": [26, 35] },
      "message":  "总镜头数 39 超出容忍上限 35（+12.9%）"
    },
    {
      "code":     "psychology_group_synonym_fallback",
      "severity": "info",
      "block_id": "B03",
      "actual":   "reversal",
      "expected": { "mapped_to": "payoff" },
      "message":  "EditMap 使用自由词 reversal，映射到 payoff"
    }
  ]
}
```

**回归阈值**（3 套剧本合计）：

- `severity: "warn"` 条数 ≤ 剧本数 × 3 = 9 条（含 S10/S11/S12 治本类告警一起计数）。
- `severity: "info"` 不设上限（运营信号）。
- 任一**硬门副告警**（`avsplit_format_failed / bgm_name_leak`）出现但 H5 却通过 → **红线**（说明 LLM 欺骗）。
- **S10 `satisfaction_subject_misaligned` 单套剧本出现 ≥ 2 次** → 红线（说明 EditMap 对 satisfaction 主体判定系统性错位，建议立即回滚剧本重写，非修代码）。
- **S12 `payoff_without_protagonist_reaction` 任一 payoff block 命中** → **必须人工复核 Director 对应 block 画面**，不得无感通过。
