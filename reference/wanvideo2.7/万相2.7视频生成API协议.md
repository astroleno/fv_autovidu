## 文生视频

万相文生视频模型基于文本提示词，生成一段流畅的视频。

### 适用范围

为确保调用成功，请务必保证模型、Endpoint URL 和 API Key 均属于同一地域。跨地域调用将会失败。

- **选择模型**：确认模型所属的地域
- **选择 URL**：选择对应的地域 Endpoint URL，支持 HTTP URL 或 DashScope SDK URL
- **配置 API Key**：选择地域并获取 API Key，再配置 API Key 到环境变量

> 本文的示例代码适用于北京地域。

### HTTP 调用（wan2.7 模型）

> **重要**：此接口为新版协议，仅支持 wan2.7 模型。

由于文生视频任务耗时较长（通常为1-5分钟），API 采用异步调用。整个流程包含 **"创建任务 -> 轮询获取"** 两个核心步骤：

#### 步骤1：创建任务获取任务ID

| 地域 | Endpoint URL |
|------|-------------|
| 北京 | `POST https://dashscope.aliyuncs.com/api/v1/services/aigc/video-generation/video-synthesis` |
| 新加坡 | `POST https://dashscope-intl.aliyuncs.com/api/v1/services/aigc/video-generation/video-synthesis` |

> 创建成功后，使用接口返回的 `task_id` 查询结果，task_id 有效期为 24 小时。请勿重复创建任务，轮询获取即可。

#### 请求头（Headers）

| 参数 | 类型 | 必选 | 说明 |
|------|------|------|------|
| Content-Type | string | 是 | 请求内容类型。此参数必须设置为 `application/json`。 |
| Authorization | string | 是 | 请求身份认证。接口使用阿里云百炼 API-Key 进行身份认证。示例值：`Bearer sk-xxxx`。 |
| X-DashScope-Async | string | 是 | 异步处理配置参数。HTTP 请求只支持异步，必须设置为 `enable`。 |

> **重要**：缺少此请求头将报错："current user api does not support synchronous calls"。

#### 请求体（Request Body）

```json
{
    "model": "wan2.7-t2v",
    "input": {
        "prompt": "一段紧张刺激的侦探追查故事，展现电影级叙事能力。第1个镜头[0-3秒] 全景：雨夜的纽约街头，霓虹灯闪烁，一位身穿黑色风衣的侦探快步行走。"
    },
    "parameters": {
        "resolution": "720P",
        "ratio": "16:9",
        "prompt_extend": true,
        "watermark": true,
        "duration": 15
    }
}
```

#### input 对象

| 参数 | 类型 | 必选 | 说明 |
|------|------|------|------|
| prompt | string | 是 | 文本提示词。用来描述生成视频中期望包含的元素和视觉特点。**支持中英文**，每个汉字/字母占一个字符，超过部分会自动截断。wan2.7-t2v 不超过5000个字符。 |
| negative_prompt | string | 否 | 反向提示词，用来描述不希望在视频画面中看到的内容，可以对视频画面进行限制。支持中英文，长度不超过500个字符。 |
| audio_url | string | 否 | 音频文件URL，模型将使用该音频生成视频。支持格式：wav、mp3；时长：2～30s；文件大小：不超过15MB。 |

#### parameters 对象

| 参数 | 类型 | 必选 | 说明 |
|------|------|------|------|
| resolution | string | 否 | 生成视频的分辨率档位。wan2.7-t2v：可选值：720P、1080P。**默认值为 1080P**。重要：resolution 直接影响费用，同一模型：1080P > 720P。 |
| ratio | string | 否 | 生成视频的宽高比。可选值：`16:9`（默认值）、`9:16`、`1:1`、`4:3`、`3:4`。 |
| duration | integer | 否 | 生成视频的时长，单位为秒。wan2.7-t2v：取值为[2, 15]之间的整数。**默认值为5**。 |
| prompt_extend | boolean | 否 | 是否开启 prompt 智能改写。`true`：默认值，开启智能改写；`false`：不开启智能改写。 |
| watermark | boolean | 否 | 是否添加水印标识，水印位于视频右下角，文案固定为"AI生成"。`false`：默认值，不添加水印；`true`：添加水印。 |
| seed | integer | 否 | 随机数种子，取值范围为 [0, 2147483647]。未指定时，系统自动生成随机种子。 |

#### 分辨率对照表

| 分辨率档位 | 宽高比 | 输出视频分辨率（宽*高） |
|-----------|--------|----------------------|
| 720P | 16:9 | 1280*720 |
| 720P | 9:16 | 720*1280 |
| 720P | 1:1 | 960*960 |
| 720P | 4:3 | 1104*832 |
| 720P | 3:4 | 832*1104 |
| 1080P | 16:9 | 1920*1080 |
| 1080P | 9:16 | 1080*1920 |
| 1080P | 1:1 | 1440*1440 |
| 1080P | 4:3 | 1648*1248 |
| 1080P | 3:4 | 1248*1648 |

#### 步骤2：根据任务ID查询结果

| 地域 | Endpoint URL |
|------|-------------|
| 北京 | `GET https://dashscope.aliyuncs.com/api/v1/tasks/{task_id}` |
| 新加坡 | `GET https://dashscope-intl.aliyuncs.com/api/v1/tasks/{task_id}` |

- `PENDING`：任务排队中

- `FAILED`：任务执行失败
- `CANCELED`：任务已取消
- `UNKNOWN`：任务不存在或状态未知

**成功响应示例**：
```json

    "output": {

        "orig_prompt": "原始prompt",
```

### 多镜头叙事

通过 `prompt` 自然语言控制镜头结构，无需配置 `shot_type` 参数：
- **单镜头**：输入"生成单镜头视频"
- **多镜头**：输入"生成多镜头视频" 或使用时间戳描述分镜（如 "第1个镜头[0-3秒] 全景：雨夜的纽约街头"）
- **默认**：未指定时，模型根据 `prompt` 内容自行理解

### 常见问题

**Q: 从 wan2.6 升级到 wan2.7，代码需要改哪些地方？**

A: 主要有两处变更：
1. **输出视频分辨率的控制方式**：wan2.7 不再使用 `size` 字段，改为通过 `resolution`（分辨率档位）和 `ratio`（宽高比）两个字段组合来定义输出视频分辨率
2. **镜头类型字段移除**：wan2.7 不再支持 `shot_type` 字段，该参数已弃用，可直接通过 prompt 来描述镜头类型

**Q: 如何将临时的视频链接转为永久链接？**

A: 不能直接转换该链接。正确的做法是：后端服务获取到 url 后，通过代码下载该视频文件，然后将其上传到永久对象存储服务（如阿里云 OSS），生成一个新的、永久访问链接。

## 图生视频

万相-图生视频模型全新升级，支持多模态输入（文本/图像/音频/视频），可完成**首帧生视频、首尾帧生视频、视频续写**三大任务。

### 模型概览

| 模型名称 | 能力支持 | 输入模态 | 输出视频规格 |
|----------|----------|----------|--------------|
| wan2.7-i2v（推荐） | 有声视频、首帧生视频、首尾帧生视频、视频续写、视频续写+尾帧控制、多镜头叙事、声画同步 | 文本、图像、音频、视频 | 分辨率档位：720P、1080P；视频时长：[2s, 15s]（整数）；固定规格：30fps、MP4 (H.264编码) |

### HTTP 调用

> **重要**：此接口为图生视频新版协议，仅支持 wan2.7 模型。

#### 步骤1：创建任务获取任务ID

| 地域 | Endpoint URL |
|------|-------------|
| 北京 | `POST https://dashscope.aliyuncs.com/api/v1/services/aigc/video-generation/video-synthesis` |
| 新加坡 | `POST https://dashscope-intl.aliyuncs.com/api/v1/services/aigc/video-generation/video-synthesis` |

#### 首帧生视频

基于首帧图像和音频生成视频。

```json
{
    "model": "wan2.7-i2v",
    "input": {
        "prompt": "一只小猫在草地上奔跑",
        "media": [
            {
                "type": "first_frame",
                "url": "https://xxx/xxx.png"
            },
            {
                "type": "driven_audio",
                "url": "https://xxx/xxx.mp3"
            }
        ]
    },
    "parameters": {
        "resolution": "720P",
        "duration": 10,
        "prompt_extend": true,
        "watermark": true
    }
}
```

#### 首尾帧生视频

传入首帧和尾帧生成视频。

```json
{
    "model": "wan2.7-i2v",
    "input": {
        "prompt": "写实风格,一只小黑猫好奇地仰望天空",
        "media": [
            {
                "type": "first_frame",
                "url": "https://xxx/first_frame.png"
            },
            {
                "type": "last_frame",
                "url": "https://xxx/last_frame.png"
            }
        ]
    },
    "parameters": {
        "resolution": "720P",
        "duration": 10,
        "prompt_extend": false,
        "watermark": true
    }
}
```

#### 视频续写

基于首段视频片段，让模型生成后续内容。

```json
{
    "model": "wan2.7-i2v",
    "input": {
        "prompt": "一只戴着墨镜的狗在街道上滑滑板，3D卡通。",
        "media": [
            {
                "type": "first_clip",
                "url": "https://xxx/video_extension_1.mp4"
            }
        ]
    },
    "parameters": {
        "resolution": "720P",
        "duration": 10,
        "prompt_extend": true,
        "watermark": true
    }
}
```

#### media 数组支持的 type 类型

| type | 说明 |
|------|------|
| first_frame | 首帧 |
| last_frame | 尾帧 |
| driven_audio | 驱动音频 |
| first_clip | 首视频片段 |

#### 支持的素材组合

- **首帧生视频**：`first_frame` 或 `first_frame` + `driven_audio`
- **首尾帧生视频**：`first_frame` + `last_frame` 或 `first_frame` + `last_frame` + `driven_audio`
- **视频续写**：`first_clip` 或 `first_clip` + `last_frame`

#### 素材限制

| 类型 | 限制 |
|------|------|
| 首帧图像 | 最多传入1张 |
| 参考图像和参考视频 | 至少传入1个，图像 + 视频 ≤ 5 |
| 参考素材主体 | 仅包含单一角色 |

#### 图像限制

- 格式：JPEG、JPG、PNG（不支持透明通道）、BMP、WEBP
- 分辨率：宽和高的范围为[240, 8000]像素
- 宽高比：1:8～8:1
- 文件大小：不超过20MB

#### 视频限制

- 格式：mp4、mov
- 时长：2～10s
- 分辨率：宽和高的范围为[240, 4096]像素
- 宽高比：1:8～8:1
- 文件大小：不超过100MB

#### 音频限制

- 格式：wav、mp3
- 时长：2～30s
- 文件大小：不超过15MB

#### parameters 对象

| 参数 | 类型 | 必选 | 说明 |
|------|------|------|------|
| resolution | string | 否 | 生成视频的分辨率档位。wan2.7-i2v：可选值：720P、1080P。**默认值为 1080P**。 |
| duration | integer | 否 | 生成视频的时长。wan2.7-i2v：取值为[2, 15]之间的整数。**默认值为5**。 |
| prompt_extend | boolean | 否 | 是否开启 prompt 智能改写。`true`：默认值，开启智能改写；`false`：不开启智能改写。 |
| watermark | boolean | 否 | 是否添加水印标识。`false`：默认值，不添加水印；`true`：添加水印。 |
| seed | integer | 否 | 随机数种子，取值范围为 [0, 2147483647]。 |

#### 步骤2：根据任务ID查询结果

| 地域 | Endpoint URL |
|------|-------------|
| 北京 | `GET https://dashscope.aliyuncs.com/api/v1/tasks/{task_id}` |
| 新加坡 | `GET https://dashscope-intl.aliyuncs.com/api/v1/tasks/{task_id}` |

### 常见问题

**Q：如何生成特定宽高比（如3:4）的视频？**

A：输出视频的宽高比由**输入素材（首帧图像或首视频片段）**决定，但无法保证严格等于输入比例（如精确 3:4），实际输出会存在微小偏差。

**执行逻辑**：以输入图像的比例为基准比例参考，结合 `resolution` 档位的目标总像素，且视频长宽必须为 16 的倍数（视频编码规范），系统会自动微调至最接近的合法分辨率。

**实践建议**：
- 输入控制：尽量使用与目标比例一致的首帧或首视频片段作为输入
- 后期处理：如果对比例有严格要求，建议在视频生成后，使用编辑工具进行简单的裁剪或黑边填充

## 参考生视频

万相-参考生视频模型支持多模态输入，可将人或物体作为主角，生成单角色表演或多角色互动视频。模型还支持智能分镜，生成多镜头视频。

### 模型概览

| 模型名称 | 能力支持 | 输入模态 | 输出视频规格 |
|----------|----------|----------|--------------|
| wan2.7-r2v（推荐） | 有声视频、多角色互动、智能分镜、多镜头叙事 | 文本、图像、视频 | 分辨率档位：720P、1080P；视频时长：[2s, 10s]（整数）；固定规格：30fps、MP4 (H.264编码) |

### HTTP 调用（wan2.7 模型）

> **重要**：此接口为新版协议，仅支持 wan2.7 模型。

#### 步骤1：创建任务获取任务ID

| 地域 | Endpoint URL |
|------|-------------|
| 北京 | `POST https://dashscope.aliyuncs.com/api/v1/services/aigc/video-generation/video-synthesis` |
| 新加坡 | `POST https://dashscope-intl.aliyuncs.com/api/v1/services/aigc/video-generation/video-synthesis` |

#### 请求示例

**多主体参考**：
```json
{
    "model": "wan2.7-r2v",
    "input": {
        "prompt": "图片1在吃饭，视频1和图片2在旁边唱歌",
        "media": [
            {
                "type": "reference_video",
                "url": "https://xxx/wan-r2v-role1.mp4"
            },
            {
                "type": "reference_video",
                "url": "https://xxx/wan-r2v-role2.mp4"
            },
            {
                "type": "reference_image",
                "url": "https://xxx/wan-r2v-object4.png"
            }
        ]
    },
    "parameters": {
        "resolution": "720P",
        "duration": 10,
        "prompt_extend": false,
        "watermark": true
    }
}
```

**主体+音色参考**：
```json
{
    "model": "wan2.7-r2v",
    "input": {
        "prompt": "图片1在吃饭",
        "media": [
            {
                "type": "reference_video",
                "url": "https://xxx/wan-r2v-role1.mp4",
                "reference_voice": "https://xxx/voice.mp3"
            }
        ]
    },
    "parameters": {
        "resolution": "720P",
        "ratio": "16:9",
        "duration": 10,
        "prompt_extend": false,
        "watermark": true
    }
}
```

#### input 对象

| 参数 | 类型 | 必选 | 说明 |
|------|------|------|------|
| prompt | string | 是 | 文本提示词。用来描述生成视频中期望包含的元素和视觉特点。支持中英文，wan2.7-r2v 不超过5000个字符。 |
| negative_prompt | string | 否 | 反向提示词，用来描述不希望在视频画面中看到的内容。支持中英文，长度不超过500个字符。 |
| media | array | 是 | 媒体素材数组，素材包括图像、视频和音频。 |

#### media 数组

| type | 说明 |
|------|------|
| reference_image | 参考图像。提供主体角色（人物/动物/物体）和场景参考 |
| reference_video | 参考视频。提供主体角色（人物/动物/物体）和音色参考，不推荐传入空镜视频 |
| first_frame | 首帧图像。基于首帧生成视频，通常包含主体角色（人物/动物/物体） |

#### 素材限制

- 首帧图像，最多传入1张
- 参考图像和参考视频至少传入1个，图像 + 视频 ≤ 5
- 参考素材为主体角色时，仅包含单一角色

#### 角色引用说明

通过"图片1、图片2"这类标识引用参考图像中的角色，通过"视频1、视频2"这类标识引用参考视频中的角色。每个参考素材（视频或图像）仅包含单一角色。模型仅通过此方式识别参考中的角色。

#### 参考图像限制

- 格式：JPEG、JPG、PNG（不支持透明通道）、BMP、WEBP
- 分辨率：宽度和高度范围为[240, 8000]像素
- 宽高比：1:8～8:1
- 文件大小：不超过20MB

#### 参考视频限制

- 格式：mp4、mov
- 时长：1～10s
- 分辨率：宽度和高度范围为[240,4096]像素
- 宽高比：1:8～8:1
- 文件大小：不超过100MB

#### reference_voice（可选）

音频 URL。用于指定参考素材（图像/视频）中主体角色的音色。

**音频逻辑**：
- 默认行为：若 reference_video 本身包含音频，但未指定 reference_voice，默认使用视频原声
- 优先级：若同时传入 reference_video（含音频）和 reference_voice，则优先使用 reference_voice 的音色，覆盖视频原声

**音频限制**：
- 格式：wav、mp3
- 时长：1～30s
- 文件大小：不超过15MB

#### parameters 对象

| 参数 | 类型 | 必选 | 说明 |
|------|------|------|------|
| resolution | string | 否 | 生成视频的分辨率档位。wan2.7-r2v：可选值：720P、1080P。**默认值为 1080P**。 |
| ratio | string | 否 | 生成视频的宽高比。可选值：`16:9`（默认值）、`9:16`、`1:1`、`4:3`、`3:4`。未传入首帧图像时按指定的 ratio 参数生成视频；已传入首帧图像时自动忽略 ratio 参数，以首帧图像的宽高比生成近似比例的视频。 |
| duration | integer | 否 | 生成视频的时长。wan2.7-r2v：取值为[2, 10]之间的整数。**默认值为5**。 |
| watermark | boolean | 否 | 是否添加水印标识。`false`：默认值，不添加水印；`true`：添加水印。 |
| seed | integer | 否 | 随机数种子，取值范围为 [0, 2147483647]。 |

#### 分辨率对照表

| 分辨率档位 | 宽高比 | 输出视频分辨率（宽*高） |
|-----------|--------|----------------------|
| 720P | 16:9 | 1280*720 |
| 720P | 9:16 | 720*1280 |
| 720P | 1:1 | 960*960 |
| 720P | 4:3 | 1104*832 |
| 720P | 3:4 | 832*1104 |
| 1080P | 16:9 | 1920*1080 |
| 1080P | 9:16 | 1080*1920 |
| 1080P | 1:1 | 1440*1440 |
| 1080P | 4:3 | 1648*1248 |
| 1080P | 3:4 | 1248*1648 |

## 使用限制

1. **数据时效**：任务 `task_id` 和视频 `video_url` 均只保留 24 小时，过期后将无法查询或下载
2. **内容审核**：输入内容和输出视频均会经过内容安全审核，包含违规内容的请求将报错 "IPInfringementSuspect" 或 "DataInspectionFailed"
